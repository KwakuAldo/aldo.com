To Do App

December 2024 Frontend Development

Hello, I am Aldo, an intermediate JavaScript developer, and this is my first blog post! I started my web development journey sometime around April 2020, and recently embarked on a journey to build a To Do application from scratch. After going through the fundamentals of JS, I wanted to test my skills by building a light weight project to test my skills, it was a comprehensive learning experience that took me through modern web development practices. In this post, I will share my complete development process, the challenges I faced, and the lessons I learned along the way.

Project Overview

App Name: To Do App

Tech Stack: HTML5, CSS3, Vanilla JavaScript, Netlify

Features: Task management, local storage, form validation, responsive design

Live Site: todoappbyaldo.netlify.app

Github repo: https://github.com/KwakuAldo

HTML Structure and Accessibility

So I started of with a simple semantic, accessible HTML structure. I implemented several accessibility best practices:

  • ARIA Labels for screen readers
  • Semantic HTML elements
  • Proper form labeling with 'for' and 'id' attributes
  • Keyboard navigation support
  • Responsive design for all screen sizes

Building with accessibility in mind is much better and easier than retrofitting it later, it also improves the overall user experience for everyone.

Adding Functionality with JavaScript

Next I wanted to add some functionality before styling. The app needed to work at the bare bones before anything else. So a step by step approach:

  • Fetch values from the input
  • Function to add task to a list
  • Function to display task
  • Functions to edit, delete, save, and mark as complete
  • Create an array to handle the task list
  • Data persistence with local storage

Implementing Local Storage

I also learned how to use the browser's local storage to persist data.

[Insert image of local storage code]

One thing I also learned while implementing local storage is to always wrap local storage operations in try-catch blocks. Users can disable local storage or run out of space, and your app should handle these scenarios gracefully.

Form Validation

I had to implement some form validation so that the inputs were not recording empty values, past dates and characters not going over certain limits. Good validation provides clear, helpful error messages and prevents users from making mistakes, it is better to guide users than to frustrate them.

Deployment

Deploying to Netlify taught me about production considerations:

  • File structure issues, initially, my CSS and JS files weren't loading because they were in the src directory, but Netlify serves from public
  • I had to update all the file paths to work in production. I achieved this by creating a netlify.toml file for proper deployment settings

My biggest take away from this project was that what works locally doesn't always work in production. Always test your deployed application and be prepared to debug.

Core Features and Functionality

  • Task creation with title, description, due date/time, time and task priority
  • Task management (edit, delete, save, mark complete)
  • Bulk operations (mark all complete, delete all, and delete completed)
  • Form validation with error messages
  • Live counters for tasks and completed tasks
  • Data persistence with local storage
  • Accessibility features throughout

Technical Achievements

  • Semantic HTML5
  • Vanilla JavaScript
  • Error handling
  • Cross browser compatibility

Development Workflow

  • File organization and project structure
  • Development vs production builds

  • Step by step feature development

Future Improvements

While this version is fully functional, I would love to add some new features in the future:

  • Search and filter function
  • Due date and time reminders
  • Dark mode toggle
  • PWA features for offline functionality

Conclusion

Building this app was an incredible learning experience that improved my understanding of JavaScript and modern web practices. Most importantly, it showed me that building a professional quality application is achievable with dedication, proper planning and step by step problem solving. The key is to start with a solid foundation and build incrementally, testing and refining along the way.

This project has given me back confidence in my abilities as a developer especially since I haven't been consistent with my learning in recent times. I am excited to continue building and learning, applying these lessons to even more complex applications.
